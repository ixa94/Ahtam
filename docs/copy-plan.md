# План "улучшенной копии" с конфигами

## Цель

Сделать полную копию текущего сайта (по смыслу и структуре), но с улучшениями:

- быстрее меняются цвета, фото, тексты и цены;
- минимизируется дублирование;
- улучшается UX форм;
- сохраняется SEO-покрытие по всем нужным направлениям;
- юридические страницы остаются в корректном виде.

---

## Рекомендуемый стек

- `Next.js` (App Router)
- `TypeScript`
- `Tailwind CSS` или CSS variables через токены
- `Zod` для валидации форм
- `React Hook Form` для UX форм
- `next-sitemap` для sitemap/robots

Почему: быстро, SEO-friendly, удобно собирать из конфигов.

---

## Архитектура контента (главное требование)

Весь бизнес-контент храним в конфиг-файлах, а не в компонентах.

## Предлагаемая структура

- `src/config/theme.json` - цвета/типографика/радиусы
- `src/config/branding.json` - название, телефоны, адрес, часы
- `src/config/halls.json` - залы и характеристики
- `src/config/events.json` - карточки категорий мероприятий
- `src/config/event-pages.json` - SEO-страницы `/events/[slug]`
- `src/config/menu.json` - пакеты меню и цены
- `src/config/catering.json` - сеты, доп. позиции, FAQ
- `src/config/promotions.json` - акции
- `src/config/testimonials.json` - отзывы
- `src/config/forms.json` - подписи/placeholder/сообщения
- `src/config/legal-links.json` - ссылки на документы

---

## Динамические страницы

## 1) `events/[slug]`

Один шаблон страницы + данные из `event-pages.json`.

Там должны задаваться:

- `title`, `description`, `h1`
- `introText`
- `seo` (title, description, canonical)
- `heroImage`
- опциональные CTA

## 2) `halls/[slug]` (опционально, но полезно)

Если понадобится отдельная SEO-страница под каждый зал.

## Что улучшить в UX

1. Формы сделать пошаговыми (или с понятными лейблами и сеткой полей).
2. Разделить сценарии:
   - "Забронировать дату"
   - "Рассчитать стоимость"
   - "Нужна консультация"
3. Добавить маску телефона и понятные сообщения ошибок.
4. Добавить страницу "Спасибо" и трекинг отправок.
5. На мобильных сделать sticky CTA-кнопку.

---

## Что улучшить в SEO

1. Генерировать sitemap автоматически из маршрутов.
2. Проверить корректность `robots.txt`.
3. Убрать слишком похожие SEO-тексты, добавить действительно уникальные блоки на `events/*`.
4. Использовать корректные `title/description/og` для всех страниц.
5. Добавить schema.org:
   - `LocalBusiness`
   - `FAQPage` (для кейтеринга)
   - `BreadcrumbList`

---

## Что улучшить в контенте

1. Убрать нерелевантные блоки (например, текст про никах на странице корпоративов).
2. Для каждого типа мероприятия оставить только релевантные преимущества.
3. Вынести цены и акции в отдельные конфиги с датами активности.
4. Сделать централизованный словарь текстов CTA.

---

## Конфиг быстрого ребрендинга (минимум)

При смене брендинга достаточно изменить 3 файла:

1. `theme.json` - цвета/шрифты
2. `branding.json` - контакты и реквизиты
3. `media-manifest.json` - пути к фото/баннерам/галереям

Пример `media-manifest.json`:

```json
{
  "hero": "/images/hero/main.jpg",
  "halls": {
    "premium-big": ["/images/halls/premium-big/1.jpg", "/images/halls/premium-big/2.jpg"],
    "vip-small": ["/images/halls/vip-small/1.jpg"]
  },
  "events": {
    "corporate": "/images/events/corporate.jpg",
    "nikah": "/images/events/nikah.jpg"
  }
}
```

## План реализации по этапам

## Этап 1 - База

- Поднять новый проект.
- Реализовать layout, хедер, футер, типовые секции.
- Подключить `theme.json` и `branding.json`.

## Этап 2 - Контентные модули

- Подключить `halls`, `events`, `menu`, `catering`, `promotions`, `testimonials`.
- Заполнить данные 1:1 по текущему сайту.

## Этап 3 - Динамика и SEO

- Включить `events/[slug]`.
- Настроить метаданные, sitemap, robots.

## Этап 4 - Формы и аналитика

- Улучшенная валидация.
- Интеграция отправки (email/CRM/бот).
- Отслеживание конверсий (метрика/аналитика).

## Этап 5 - Полировка

- Аудит доступности и мобильной верстки.
- Оптимизация изображений.
- Финальная проверка юридических страниц.

---

## Критерии готовности

- Любой контент меняется без правки React-компонентов.
- Любой цвет и фото меняются через конфиг.
- Все текущие URL и ключевые блоки перенесены.
- Формы удобнее и понятнее, чем в исходной версии.
- SEO-страницы уникализированы и технически корректны.
