## Структура контента по типам страниц

## Главная `/`

- Полный лендинг со всеми продающими блоками.
- Содержит:
  - USP/преимущества
  - услуги (банкет/кейтеринг)
  - статистику
  - меню/кухню
  - направления мероприятий
  - залы
  - акции
  - отзывы
  - форму бронирования

## Страница залов `/hall/`

- Каталог залов с характеристиками:
  - вместимость
  - парковка
  - ценовой ориентир
- Далее снова повторяется набор типовых блоков (кухня, отзывы, форма и т.д.).

## Страница мероприятий `/events/`

- Перечень типов мероприятий.
- Видеообзор залов.
- Повторяющийся блок отзывов и форма.

## Страницы событий `/events/*`

- Уникальный intro-абзац под событие.
- Далее почти полностью общий шаблон.
- Это ядро SEO-стратегии: отдельный URL под каждый интент.

## Страница никаха `/nikah/`

- Самый релевантный контент для мусульманской аудитории:
  - раздельные залы
  - помощь с имамом
  - халяль-меню

## Страница кейтеринга `/kejtering/`

- Уникальная товарная логика:
  - сеты
  - дополнительные блюда
  - цены
  - FAQ
  - формы заявки

## Страница меню `/menyu/`

- Пакеты банкетного меню + цены.
- Детское меню.
- Продуктовые позиции (напитки, бэлеш).

## Структура форм (единая)

Типовые поля:

- дата мероприятия
- количество гостей
- выбор зала
- телефон
- согласие на обработку персональных данных

Типовые сценарии:

- забронировать
- рассчитать мероприятие
- обратный звонок

## Рекомендуемая структурная модель для новой версии

## 1) Route groups

- `/(public)/` - главная и контентные страницы
- `/(events)/` - динамические event-страницы
- `/(legal)/` - юридические документы

## 2) Динамические маршруты

- `/events/[slug]`
- `/halls/[slug]` (даже если пока все залы на одной странице)

## 3) Компонентная декомпозиция

- `Header`
- `HeroSection`
- `FeaturesGrid`
- `ServicesCards`
- `StatsSection`
- `HallsGrid`
- `TestimonialsSection`
- `FAQSection`
- `BookingForm`
- `LocationSection`
- `LegalFooter`

## 4) Контент и UI разнести

- Контент (тексты, цены, фото, списки) только в конфигах/JSON/MD.
- Компоненты не содержат хардкод бизнес-текстов.

Это позволит быстро запускать новые страницы событий и менять контент без правок шаблонного кода.
